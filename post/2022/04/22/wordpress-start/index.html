# 为什么选择 WordPress

WordPress 是一个内容管理系统 (CMS)，您可以使用它来构建和管理您的网站。第一个版本由 Matt Mullenweg 和 Mike Little 于 2003 年创建，作为名为 b2 的程序的替代版本。

从一开始，WordPress 就被设计为开源软件。我们很快就会更多地讨论这个概念，但简而言之，这意味着 WordPress 是由一个庞大的志愿者程序员和设计师社区维护的。因此，WordPress 随着时间的推移不断发展，每个版本都获得了新的特性和功能。

虽然 WordPress 最初是为博客设计的，但它现在适用于各种网站。事实上，它为互联网上超过 28%的网站提供支持。您可以使用它来创建商业网站、在线投资组合、电子商务商店等等。要更清楚地了解 WordPress 的功能，请查看一些用它制作的最大和最受欢迎的网站。

WordPress 是一个开源平台。这意味着任何人都可以免费下载该软件并对其进行更改和改进，并且它由一个蓬勃发展的专用社区维护。

由于织梦 dedecms 从免费突然开始收费授权后，对国内开源 CMS 起了一个坏头，固然还是选择国际开源项目进行使用和二次开发，WordPress 维护多年文档和插件都比较齐全，大大减少了学习的成本。

# 优点

  1. WordPress 功能强大、扩展性强，这主要得益于其插件众多，易于扩充功能，基本上一个完整网站该有的功能，通过其第三方插件都能实现所有功能；
  2. wordpress 搭建的博客对 seo 搜索引擎友好，收录也快，排名靠前；
  3. 适合 DIY，如果你是喜欢丰富内容的网站，那么 wordpress 可以很好地符合你的胃口。
  4. 主题很多，网站上一大片都是 wordpress 的主题，各色各样，应有尽有！
  5. wordpress 备份和网站转移比较方便，原站点使用站内工具导出后，使用 WordPress Importer 插件就能方便地将内容导入新网站。
  6. wordpress 有强大的社区支持，有上千万的开发者贡献和审查 wordpress，所以 wordpress 是安全并且活跃的。

# 缺点

  1. wordpress 源码系统初始内容基本只是一个框架，需要时间自己搭建；
  2. 插件虽多，但是不能安装太多插件，否则会拖累网站速度和降低用户体验； 服务器空间选择自由较小；
  3. 静态化较差，确切地说是真正静态化做得不好，如果要想对整个网站生成真正静态化页面，还做不好，最多只能生成首页和文章页静态页面，所以只能对整站实现伪静态化!
  4. wordpress 的博客程序定位，简单的数据库层等都注定了他不能适应大数据。
  5. wordpress 使用的谷歌字体、头像经常被墙，访问加载速度慢，不能一键更新。

# 运行所需要的环境

# 下载WordPress

下载wordpress

下载完成，解压 wordpress-5.9.3.zip

# MySQL 的安装

MySQL 官网下载

这里使用的是macOS。

使用终端homebrew输入命令：

brew install mysql

安装完成以后会有提示，先启动 mysql 输入命令：

mysql.server start

启动成功，即可进行初始化输入命令：

mysql_secure_installation

连接数据库，上面的设置完成以后，开始连接数据库，输入命令：

mysql -u root -p

连接成功后，我们新建一个数据库，输入命令：

create database wordpress;

# PHP 的安装

PHP 官网下载

使用homebrew安装,wordpress 使用的 PHP 版本是 7.4，我们这里选择 7.4 进行安装。

brew install php@7.4

按照提示我们新增.zshrc 文件,配置 php 目录到系统环境，注意$PATH 不要省略否则会导致系统原有配置失效

export PATH="/opt/homebrew/opt/php@7.4/bin:$PATH"

export PATH="/opt/homebrew/opt/php@7.4/sbin:$PATH"

然后让配置生效，使用终端

source .zshrc

启动php进程

brew services start php@7.4

# NGINX 的安装

nginx 官网下载

使用homebrew直接安装

brew install nginx

安装完成会提示配置目录

/opt/homebrew/etc/nginx/nginx.conf

编辑配置文件

server {
        listen       8080; # 端口号可以自己设置
        server_name  localhost;

        #charset koi8-r;

        #access_log  logs/host.access.log  main;

        location / {
            root   "/Users/marks/Downloads/wordpress/";
            index  index.html index.htm index.php;
        }

        #error_page  404              /404.html;

        # redirect server error pages to the static page /50x.html
        #
        error_page   500 502 503 504  /50x.html;
        location = /50x.html {
            root   html;
        }

        # proxy the PHP scripts to Apache listening on 127.0.0.1:80
        #
        #location ~ \.php$ {
        #    proxy_pass   http://127.0.0.1;
        #}

        # pass the PHP scripts to FastCGI server listening on 127.0.0.1:9000
        #
        location ~ \.php$ {
           root           "~/Downloads/wordpress";  # 你的wordpress目录
           fastcgi_pass   127.0.0.1:9000;
           fastcgi_index  index.php;
           fastcgi_param  SCRIPT_FILENAME  $document_root$fastcgi_script_name;
           include        fastcgi_params;
           try_files $uri $uri/ /index.php?$args;
        }

        # deny access to .htaccess files, if Apache's document root
        # concurs with nginx's one
        #
        location ~ /\.ht {
           deny  all;
        }
    }

编辑完成使用命令重新启动nginx

brew services restart nginx

打开浏览器访问 localhost:8080，出现以下页面说启动成功，然后选择语言后我们进入下一步

我们来配置数据库信息

配置wordpress站点信息，这里我作为演示使用了弱密码

自动进入登录页，使用我们设置的账户和密码进行登录

进入管理界面

访问默认连接，查看前台效果localhost:8080，到这里wordpress的安装就完成了。
